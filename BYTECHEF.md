### Project Description
`warp-prometheus` is a Rust library designed to easily add Prometheus metrics collection to applications built with the `warp` web framework. It captures HTTP request duration and status, allowing for granular monitoring of application performance and health.

### Architecture
The core of `warp-prometheus` is the `Metrics` struct, which encapsulates a Prometheus `HistogramVec`. This `HistogramVec` tracks `server_response_duration_seconds` with labels for `classifier` (derived from HTTP method and a sanitized path) and `status` (HTTP status code). The `Metrics` struct provides a `http_metrics` function, which is intended to be used with `warp`'s `log::custom` filter. This function processes `warp::log::Info` to extract request details, sanitize the request path using `sanitize_path_segments` (which generalizes path parameters into `*`), and records the elapsed time and status into the `HistogramVec`.

### How to Run
This project is a library and is not meant to be run as a standalone application. To use it:

1.  **Add as a dependency**: Include `warp-prometheus` in your `Cargo.toml`:
    ```toml
    [dependencies]
    warp-prometheus = "0.5.0" # Or the latest version
    ```
2.  **Build**: Compile the library and your dependent project:
    ```bash
    cargo build
    ```
3.  **Integrate**: Follow the examples in `README.md` or `src/lib.rs` to initialize `warp_prometheus::Metrics` and apply it to your `warp` routes.

### How to Test
The project includes unit tests for its core logic, specifically the `sanitize_path_segments` function.
To run the tests:
```bash
cargo test
```
The tests are located within the `src/lib.rs` file under the `#[cfg(test)] mod test` block.
